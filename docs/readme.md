电脑迷宫鼠（Maze Mouse）课程设计项目说明书

一、项目背景

本项目是《程序设计实践 II》课程综合实践课题，项目名称为电脑迷宫鼠（Maze Mouse）。项目以Java面向对象编程为核心，设计并实现一套可在迷宫环境中自主移动、路径搜索与最优路径计算的模拟系统。系统兼顾算法实现、可视化展示与交互操作，满足课程实践要求与答辩展示需求。

项目目标是实现具备完整迷宫处理与智能寻路能力的电脑鼠程序，能够完成迷宫自动生成、地图显示、单一路径搜索、迷宫全域遍历、最短路径计算，并以动画形式实时呈现搜索过程，直观展示搜索过程中栈、队列等数据结构的状态变化。

本项目面向课程答辩，整体设计遵循功能完整、结构清晰、界面友好、代码易于讲解的原则，确保实现逻辑、算法思路、程序架构均可清晰展示与说明。

二、技术要求

项目采用标准Java桌面程序开发，技术选型严格遵循课程约束，具体要求如下：

• 开发语言：Java 17及以上版本

• 图形界面：Java Swing，禁止使用JavaFX、Web前端框架、SpringBoot及第三方大型GUI组件库

• 开发模式：纯面向对象设计（OOP），功能模块化，职责清晰划分

• 运行形态：独立可运行的Java桌面应用程序

三、功能需求

1. 自动生成迷宫

支持基于算法的随机迷宫自动生成，优先采用Prim算法实现。每次生成的迷宫结构不同，保证起点与终点之间可达，整体布局合理且具备典型迷宫特征。系统支持重新生成，可快速刷新迷宫布局。

2. 起点与终点设置

系统提供默认起点与默认终点，满足快速运行与测试需求。同时支持扩展功能，可通过鼠标点击方式手动设置起点与终点，提升交互灵活性。

3. DFS寻路功能

实现深度优先搜索（DFS）算法，用于寻找迷宫中一条可行通路。在搜索过程中实时展示探索轨迹与回溯过程，可视化标记当前访问节点与回退节点，清晰体现DFS的搜索逻辑。

4. BFS最短路径

实现广度优先搜索（BFS）算法，用于计算从起点到终点的最短路径。搜索完成后显示路径长度、访问节点总数等信息，直观体现BFS在最短路径问题上的优势。

5. 实时动画回放

系统支持搜索过程动画回放，实现节点逐步访问、路径逐步绘制、回溯过程动态展示。支持播放速度调节，动画采用Swing Timer实现，不使用Thread.sleep阻塞界面线程，保证界面流畅。

6. 搜索日志面板

界面设置专用日志区域，以时间戳形式实时输出搜索状态，包括搜索启动、节点访问、死路判断、回溯执行、终点找到等关键信息，日志内容自动刷新，便于观察程序运行状态。

7. 数据结构状态展示

界面提供运行状态面板，实时显示当前运行算法名称、栈（Stack）大小、队列（Queue）大小、已访问节点数量、当前路径长度、运行耗时等信息，直观呈现数据结构在搜索过程中的变化。

四、数据结构要求

项目必须使用课程指定数据结构，清晰体现数据结构应用能力：

• 迷宫存储：使用二维数组Cell[][] maze保存迷宫单元格信息

• DFS实现：使用Stack<Cell>存储搜索路径，支持回溯操作

• BFS实现：使用Queue<Cell>（基于LinkedList）实现层序扩展，完成最短路径搜索

• 路径恢复：通过父节点记录机制（Map<Cell, Cell>或Cell.parent），从终点反向回溯至起点，还原完整路径

五、界面设计

主窗口布局

整体界面采用简洁清晰的布局，便于操作与展示，布局结构如下：

• 顶部区域：迷宫图形显示面板

• 中部区域：功能按钮栏，包含生成迷宫、DFS寻路、BFS最短路径、动画回放、暂停、重置

• 下部区域：状态栏与日志面板

界面要求

布局简洁合理，视觉效果整洁美观，支持窗口缩放，组件大小自适应，操作逻辑直观易懂。

六、面向对象设计要求

项目采用职责单一的模块化类设计，推荐核心类如下：

• MainFrame：主窗口，统筹界面与功能调度

• Maze：迷宫数据管理类，维护迷宫整体信息

• Cell：迷宫单元格类，封装坐标、墙、访问状态、路径标记、父节点等属性

• MazeGenerator：迷宫生成器，实现Prim算法生成迷宫

• DFSPathFinder：DFS寻路算法实现类

• BFSPathFinder：BFS最短路径算法实现类

• AnimationPlayer：动画控制器，管理搜索过程动画

• MazePanel：迷宫绘制组件，负责图形渲染

• LogPanel：日志面板，输出运行信息

• StatusPanel：状态面板，展示数据结构与运行状态

七、代码规范

• 类、变量、方法命名规范，见名知意

• 类职责单一，结构清晰，降低耦合

• 代码注释充分，注释行数不少于总代码量的三分之一

• 优先保证代码可读性、可维护性与答辩可解释性，便于讲解与理解

八、答辩重点

答辩时需清晰阐述以下核心内容：

1. Prim算法生成迷宫的基本原理与执行流程

2. DFS算法使用栈（Stack）的原因与作用

3. BFS算法使用队列（Queue）的原因与作用

4. BFS能够保证最短路径的理论依据

5. 动画实现机制与线程安全处理方式

6. 项目面向对象设计思路与架构优势

7. 各模块类的功能与相互调用关系

九、扩展功能（加分项）

在完成基础功能前提下，可选择实现以下扩展功能提升项目评分：

1. AI寻路解说，实时输出电脑鼠行为描述

2. 搜索速度动态调节功能

3. 支持多种迷宫尺寸选择

4. 支持多种迷宫生成算法切换

5. 迷宫界面图片导出功能

6. 深色模式/浅色模式切换

十、开发原则

项目开发遵循以下优先级：
功能正确 ＞ 代码清晰 ＞ 界面美观 ＞ 高级设计模式
