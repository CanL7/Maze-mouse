# 电脑迷宫鼠答辩材料

## 一、项目概述

本项目名称为“电脑迷宫鼠（Maze Mouse）”，是一个基于 Java Swing 的桌面程序。

项目目标是完成 4 件核心事情：

1. 使用 Prim 算法随机生成迷宫。
2. 使用 DFS 算法寻找一条可行路径。
3. 使用 DFS + 回溯遍历起点到终点的所有可行路径。
4. 根据所有可行路径的遍历结果，找出最短路径。

除了算法本身，项目还配套实现了动画回放、日志面板、状态面板和迷宫文件导入导出，方便展示算法执行过程。

## 二、技术选型

- 开发语言：Java 17+
- 图形界面：Swing
- 核心数据结构：`Cell[][]`、`Stack<Cell>`、`Queue<Cell>`
- 动画机制：`javax.swing.Timer`

这样选型的原因是完全符合课程要求，而且便于答辩时从数据结构和面向对象两个角度展开说明。

## 三、系统功能

项目当前支持以下功能：

- 生成迷宫：随机生成一个 `20 x 20` 的可达迷宫。
- DFS 寻路：找到一条从起点到终点的可行路径，并展示探索与回溯。
- 所有可行路径：使用 DFS + 回溯遍历起点到终点的全部通路。
- BFS 最短路径：计算从起点到终点的最短路径，并统计访问节点数和路径长度。
- 动画回放：将搜索或遍历过程逐步播放出来。
- 暂停与调速：支持暂停、继续和速度调节。
- 日志显示：实时输出每一步访问、回溯和终点到达信息。
- 状态显示：显示当前算法、已访问节点数、路径长度、Stack 和 Queue 的变化。
- 文件导入导出：支持 `.maze` 格式迷宫文件保存和加载。

## 四、系统结构

项目整体采用分层设计，主要分为 4 层：

1. 入口层

## 五、核心数据结构设计

### 1. Cell[][]

整个迷宫使用二维数组 `Cell[][]` 存储。

每个 `Cell` 保存：

- 行列坐标
- 四面墙信息
- 是否访问过
- 是否为最终路径
- 是否发生回溯
- 父节点

这样设计的好处是结构清晰，既能支持迷宫生成，也能支持 DFS、BFS 和动画显示。

### 2. Stack<Cell>

`Stack<Cell>` 用在两个地方：

- DFS 寻路
- 所有可行路径遍历

原因是深度优先搜索天然符合“后进先出”的特点。进入新节点时压栈，走到死路时弹栈回溯，因此能直观体现 DFS 的搜索过程。

### 3. Queue<Cell>

`Queue<Cell>` 用于 BFS 最短路径。

原因是广度优先搜索需要按层扩展节点，先进先出正好保证先访问距离起点更近的节点，因此第一次到达终点时一定是最短路径。

## 六、核心算法说明

### 1. Prim 迷宫生成

实现思路：

1. 初始状态下每个格子四周都有墙。
2. 从起点开始，把相邻墙加入候选集合。
3. 随机选一堵墙，如果对面的格子还没访问过，就打通这堵墙。
4. 将新格子的相邻墙继续加入候选集合。
5. 重复直到所有格子都连通。

这个算法可以生成随机、连通、无环的迷宫。

### 2. DFS 寻路

实现思路：

1. 起点入栈并标记访问。
2. 取栈顶节点，寻找一个未访问的可通行邻居。
3. 如果找到邻居，就继续深入。
4. 如果找不到邻居，就说明走到死路，执行回溯。
5. 找到终点后停止，并根据父节点恢复一条可行路径。

特点：

- 能找到一条路径，但不一定最短。
- 动画上最适合展示“探索”和“回溯”。

### 3. 所有可行路径遍历

实现思路和 DFS 寻路类似，但有两个关键区别：

- DFS 寻路在到达终点时停止。
- 所有可行路径遍历在到达终点时会记录当前路径，但不会停止。
- 回溯后还会继续搜索其他分支，直到所有从起点到终点的可行路径都被找出来。

这个功能主要用来满足任务书中的“遍历迷宫中所有可行路径”要求。遍历结束后，程序还会把所有通路中最短的一条高亮出来。

### 4. BFS 最短路径

实现思路：

1. 起点入队并标记访问。
2. 不断从队首取节点。
3. 将所有未访问的可通行邻居加入队列，并记录父节点。
4. 第一次访问到终点时立即停止。
5. 从终点沿父节点反向恢复整条路径。

特点：

- 一定能找到最短路径。
- 适合展示 Queue 的层序扩展特性。

## 七、动画实现机制

本项目没有使用 `Thread.sleep()`，而是使用 `Swing Timer`。

实现方式如下：

1. 算法先运行，生成一组 `SearchStep`。
2. `AnimationPlayer` 使用 `Timer` 按顺序取出每一步。
3. 每播放一步，就更新单元格状态。
4. 然后重绘迷宫，并同步刷新日志和状态面板。

这样做的优点是不会阻塞界面线程，动画更流畅，也更符合 Swing 的线程安全要求。

## 九、可直接使用的答辩讲稿

大家好，我的课程设计项目名称是“电脑迷宫鼠”。这是一个基于 Java Swing 开发的桌面程序，主要目标是把迷宫生成、深度优先搜索、广度优先搜索和动画演示结合起来，做成一个既能运行、又便于观察算法过程的可视化系统。

在功能上，这个项目完成了 4 个核心算法任务。第一是使用 Prim 算法随机生成迷宫；第二是使用 DFS 找到从起点到终点的一条可行路径；第三是使用 DFS 加回溯遍历从起点到终点的所有可行路径；第四是根据遍历结果和 BFS 计算出从起点到终点的最短路径。除此之外，我还实现了动画回放、日志显示、状态面板以及迷宫文件导入导出功能。

在数据结构方面，整个迷宫使用 `Cell[][]` 二维数组保存。每个格子对象 `Cell` 中保存坐标、墙体状态、访问状态、路径标记和父节点等信息。DFS 寻路和所有可行路径遍历都使用 `Stack<Cell>`，因为深度优先搜索本身就是一个不断深入、再回退的过程，和栈的后进先出特性非常匹配。BFS 最短路径使用 `Queue<Cell>`，因为队列先进先出，可以实现按层扩展，所以第一次到达终点时一定就是最短路径。

在界面设计上，主窗口分成迷宫显示区、按钮控制区、状态面板和日志面板。用户可以先生成迷宫，也可以导入迷宫，然后选择执行 DFS 寻路、所有可行路径或者 BFS 最短路径。算法完成后，界面会先显示结果，再通过动画逐步回放搜索过程。动画部分使用的是 `Swing Timer`，而不是 `Thread.sleep()`，这样可以避免阻塞界面线程，保证动画流畅。

这个项目的一个重点是把算法运行过程可视化。例如，蓝色表示访问过的节点，橙色表示回溯节点，黄色表示最终路径。日志面板会记录每一步访问和回溯，状态面板会显示当前算法、访问节点数、路径长度以及栈和队列的变化，这样可以更直观地展示数据结构在算法中的作用。

总的来说，这个项目不仅实现了课程要求的迷宫生成、单路径搜索、所有可行路径遍历和最短路径功能，还通过动画和状态展示提升了可解释性，比较适合作为一个数据结构和面向对象结合的课程设计项目。

## 十、常见答辩问题与参考回答

### 1. 为什么 DFS 不一定是最短路径？

因为 DFS 的目标是优先向一条路不断深入，它只保证能找到一条可行路径，但不会比较所有路径长度，所以不一定最短。

### 2. 为什么 BFS 能保证最短路径？

因为 BFS 按层遍历，从起点出发先访问距离为 1 的节点，再访问距离为 2 的节点。第一次到达终点时，所经过的边数一定最少，因此路径最短。

### 3. 为什么“所有可行路径”要单独实现，而不是直接用 DFS 寻路代替？

因为 DFS 寻路在找到终点后就会停止，它只能得到一条可行路径；而“所有可行路径”要求在找到一条路后继续回溯和搜索其他分支，直到所有通路都找出来，所以两者目标不同。为了让功能更清晰、答辩更容易说明，我把它们拆成了两个独立模块。

### 4. 为什么动画用 Swing Timer？

因为 `Swing Timer` 在事件派发线程中调度界面刷新，不会像 `Thread.sleep()` 一样阻塞主界面，更适合做 Swing 动画。
